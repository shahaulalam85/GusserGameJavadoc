## 3. Best Repos to Actually Contribute To (beginner-friendly, real PR practice)

These are chosen for having active maintainers, clear contribution docs, and a steady stream of labeled easy issues — better first targets than mega-projects like PyTorch.

| Repo | Link | Why it's a good first contribution target |
|---|---|---|
| `firstcontributions/first-contributions` | https://github.com/firstcontributions/first-contributions | Built specifically to practice the fork → branch → PR flow risk-free |
| `scikit-learn/scikit-learn` | https://github.com/scikit-learn/scikit-learn/issues?q=is%3Aissue+is%3Aopen+label%3A%22good+first+issue%22 | Has a dedicated "good first issue" label and detailed contributing guide; Python, matches your skill level |
| `public-apis/public-apis` | https://github.com/public-apis/public-apis | Easy entry point — add/update an API listing, low code complexity |
| `EbookFoundation/free-programming-books` | https://github.com/EbookFoundation/free-programming-books | Doc-only contributions, very low barrier, still counts as real OSS history |
| `streamlit/streamlit` | https://github.com/streamlit/streamlit | You already know Streamlit from your BRAIN pipeline — real edge here |
| Any repo tagged "good first issue" | https://github.com/topics/good-first-issue | Always has fresh, currently-open beginner issues across all languages |

## How to use this for your plan

- **Days 1–7 (Git/OSS block):** browse `firstcontributions/first-contributions` to practice the PR flow, then shortlist 2–3 repos from Section 3.
- **Days 8–15:** open your first real PR — `scikit-learn` or `public-apis` are the safest bets given your current Python level.
- **Days 16–22:** since your capstone should lean on your NewsFlow/Streamlit background, `streamlit/streamlit`'s issue tracker is worth checking specifically — contributing to a tool you already use in production gives you the most leverage.

**Note on freshness:** GitHub star counts and trending repos shift constantly — the "AI Repos" section especially can look different in a few months. Worth re-checking `github.com/trending` and `EvanLi/Github-Ranking` before each contribution round rather than treating this list as permanent.
